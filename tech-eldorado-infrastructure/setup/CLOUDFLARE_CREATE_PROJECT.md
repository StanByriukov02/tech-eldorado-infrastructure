# КАК СОЗДАТЬ ПРОЕКТ В CLOUDFLARE PAGES

**ПРОБЛЕМА:** Вижу "No projects found" на странице Workers & Pages

═══════════════════════════════════════════════════════════════════════════════

## 🚀 ЧТО ДЕЛАТЬ СЕЙЧАС

### Ты на странице "Workers & Pages"

**Видишь:**
- "No projects found"
- "You have not created any projects yet."

**Нужно создать проект!**

═══════════════════════════════════════════════════════════════════════════════

## 📋 ШАГ 1: СОЗДАТЬ ПРОЕКТ

### 1.1. Нажать "Create application"

```
Вверху справа на странице "Workers & Pages":
- Видишь кнопку "Create application" (синяя кнопка)
- Нажать на неё
```

### 1.2. Выбрать тип проекта

```
После нажатия "Create application" откроется меню:
- Выбрать "Pages" (или "Pages project")
- НЕ выбирать "Worker"!
```

### 1.3. Подключить GitHub

```
1. Выбрать "Connect to Git"
2. Выбрать "GitHub"
3. Авторизовать Cloudflare в GitHub (если ещё не авторизован)
4. Выбрать репозиторий: tech-eldorado-infrastructure
5. Нажать "Begin setup"
```

═══════════════════════════════════════════════════════════════════════════════

## 📋 ШАГ 2: НАСТРОИТЬ BUILD SETTINGS

### После выбора репозитория откроется страница настройки:

```
1. Project name:
   Ввести: tech-eldorado
   
2. Production branch:
   Выбрать: main (или master)
   
3. Build command:
   Ввести: npm run build
   
4. Build output directory:
   Ввести: dist
   
5. Root directory (опционально):
   Ввести: frontend
   ИЛИ оставить пустым
```

═══════════════════════════════════════════════════════════════════════════════

## 📋 ШАГ 3: ДОБАВИТЬ ENVIRONMENT VARIABLES

### На той же странице настройки:

```
1. Прокрутить вниз до секции "Environment variables"
2. Нажать "Add variable" или "+"
3. Добавить переменные:

   Name: SUPABASE_URL
   Value: https://kgregicsrvqrndublgmp.supabase.co
   
   Name: SUPABASE_ANON_KEY
   Value: [скопировать из .env файла]
   
   Name: VITE_SUPABASE_URL
   Value: https://kgregicsrvqrndublgmp.supabase.co
   
   Name: VITE_SUPABASE_ANON_KEY
   Value: [скопировать из .env файла]
```

**Где взять SUPABASE_ANON_KEY:**
- Открыть файл `.env` в проекте
- Найти строку: `SUPABASE_ANON_KEY=...`
- Скопировать значение (без SUPABASE_ANON_KEY=)

═══════════════════════════════════════════════════════════════════════════════

## 📋 ШАГ 4: СОХРАНИТЬ И DEPLOY

```
1. После заполнения всех полей нажать "Save and Deploy" (внизу)
2. Дождаться первого deploy (~2-3 минуты)
3. После успешного deploy увидишь URL: https://tech-eldorado.pages.dev
```

═══════════════════════════════════════════════════════════════════════════════

## ⚠️ ВАЖНО

**Если frontend ещё не создан:**
- Deploy может упасть с ошибкой
- Это нормально! Проект всё равно создан
- Можно настроить позже когда frontend будет готов

**Главное сейчас:**
- Проект создан
- Репозиторий подключён
- Build settings настроены
- Environment variables добавлены
