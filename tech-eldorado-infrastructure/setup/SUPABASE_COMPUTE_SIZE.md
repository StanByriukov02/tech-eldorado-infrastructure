# ВЫБОР COMPUTE SIZE В SUPABASE

**ВОПРОС:** Какой compute size выбирать для Tech Eldorado?

═══════════════════════════════════════════════════════════════════════════════

## 🎯 РЕКОМЕНДАЦИЯ: НАЧАТЬ С МИНИМАЛЬНОГО (Micro/Small)

### Почему минимальный?

**ФАКТОРЫ:**
1. **Бюджет критичен** (~$1000 total, $25/month уже на Pro Plan)
2. **Начало проекта** - нагрузка будет расти постепенно
3. **Можно масштабировать позже** - Supabase позволяет upgrade без потери данных
4. **22 агента НЕ работают 24/7 сразу** - нагрузка распределена
5. **Database queries простые** - в начале не будет сложных запросов

**СТРАТЕГИЯ:**
- Начать с минимального (дешёвого)
- Мониторить производительность
- Upgrade когда нужно (когда увидим bottleneck)

═══════════════════════════════════════════════════════════════════════════════

## 📊 ОПЦИИ COMPUTE SIZE (обычно в Supabase Pro Plan)

### Вариант 1: Micro (самый дешёвый) ✅ РЕКОМЕНДУЕТСЯ ДЛЯ СТАРТА

```
Характеристики:
- 0.5 GB RAM
- 0.25 vCPU
- ~$25/month (базовая цена Pro Plan)

ПОДХОДИТ ДЛЯ:
✅ Начало проекта
✅ Небольшая нагрузка (первые недели)
✅ Тестирование
✅ Бюджет-оптимизация

НЕ ПОДХОДИТ ДЛЯ:
❌ Высокая нагрузка (1000+ запросов/сек)
❌ Сложные queries
❌ Real-time с множеством подключений
```

### Вариант 2: Small (средний)

```
Характеристики:
- 1 GB RAM
- 0.5 vCPU
- ~$35-40/month (доплата к Pro Plan)

ПОДХОДИТ ДЛЯ:
✅ Средняя нагрузка
✅ 10-20 агентов работают одновременно
✅ Real-time координация

КОГДА ВЫБРАТЬ:
- Если Micro недостаточно (после тестирования)
- Если есть бюджет
```

### Вариант 3: Medium/Large (самый мощный)

```
Характеристики:
- 2-4 GB RAM
- 1-2 vCPU
- ~$50-80/month (доплата к Pro Plan)

ПОДХОДИТ ДЛЯ:
✅ Высокая нагрузка
✅ Все 22 агента работают одновременно
✅ Сложные queries
✅ Production нагрузка

КОГДА ВЫБРАТЬ:
- Только если Micro/Small недостаточно
- Когда система в production
- Когда есть бюджет
```

## 🎯 РЕКОМЕНДАЦИЯ ДЛЯ TECH ELDORADO

### НАЧАТЬ С: Micro (минимальный) ✅

**ПОЧЕМУ:**
1. **Бюджет:** $25/month vs $50-80/month = экономия $25-55/мес
2. **Достаточно для старта:** Первые недели нагрузка будет низкой
3. **Можно upgrade:** Supabase позволяет изменить compute size без потери данных
4. **Мониторинг:** Следить за производительностью, upgrade когда нужно

### КОГДА UPGRADE:

**Upgrade до Small если:**
- Видишь медленные queries (>1 секунда)
- Database CPU > 80% постоянно
- Real-time соединения падают
- Агенты работают медленно

**Upgrade до Medium если:**
- Small недостаточно
- Все 22 агента работают одновременно
- Production нагрузка
- Partnership демо готово (критично!)
